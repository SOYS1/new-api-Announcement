# design-system

## 本目录用途

存放项目的设计系统文档与页面规范，用于统一视觉、组件和交互规则。

## 目录结构

```text
design-system/
└─ newapi-公告生成器/
   ├─ MASTER.md
   └─ pages/
```

## 文件/子目录说明

| 名称 | 类型 | 作用 |
|------|------|------|
| `newapi-公告生成器/` | 目录 | `newapi-Announcement` 项目的设计规范容器 |
| `newapi-公告生成器/MASTER.md` | 文件 | 全局设计规则主文件（颜色、字体、组件规范、反模式、交付检查项） |
| `newapi-公告生成器/pages/` | 目录 | 页面级规则目录；若存在对应页面文件，应优先覆盖 `MASTER.md` 的通用规则 |

## 约定与注意事项

- `MASTER.md` 明确约定：构建具体页面时先查 `pages/[page-name].md`；若不存在再回退到主规则。
- 目前 `pages/` 目录为空，新增页面规则时建议按页面名称创建同名 Markdown。
- 本目录仅存放设计文档，不包含可执行代码与运行依赖。
